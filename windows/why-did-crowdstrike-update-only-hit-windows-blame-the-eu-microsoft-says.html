Why Did CrowdStrike Update Only Hit Windows? Blame the EU, Microsoft Says

22 July, 2024

Friday's CrowdStrike update nightmare took down 8.5 million Windows-based machines globally, prompting some to question why Microsoft allows third-party kernel drivers while Apple does not. According to a Microsoft exec, you can thank the EU.

As Microsoft's Chief Communications Officer, Frank X. Shaw, noted on X, a 2009 agreement between the European Commission and Microsoft required Redmond to give security software the same level of access to Windows as Microsoft itself.

The agreement says: "Microsoft shall ensure that third-party software products can interoperate with Microsoft’s Relevant Software Products using the same Interoperability Information on an equal footing as other Microsoft Software Products.

"Microsoft shall make available to interested undertakings Interoperability Information that enables non-Microsoft server Software Products to interoperate with Windows Server Operating System on an equal footing with other Microsoft Server Software Products," it adds.

Such access is as a double-edged sword. Security software vendors can make products that take full advantage of Windows, which is great, until you have a buggy update that gets pushed to millions of Windows devices.

CrowdStrike Founder and CEO George Kurtz blamed the outage on "a defect found in a Falcon content update for Windows hosts."

Shaw took to X to take issue with the headline of a Wall Street Journal article about the outage, which says, "Blue Screens Everywhere Are Latest Tech Woe for Microsoft." Microsoft reps have argued that it's not fair to refer to Friday's incident as a "Windows outage," and while it was indeed a CrowdStrike problem, the dreaded "blue screen of death" on Windows machines was the first clue that something was wrong that morning.

Shaw also criticized the sub-head, which referenced "Microsoft’s security challenges." In May, after Chinese and then Russian hackers breached the company’s systems, Microsoft said it was "making security our top priority at Microsoft, above all else—over all other features."

"It had nothing to do with security challenges, other than CrowdStrike is a security ISV [independent software vendor]," Shaw says. "If the same thing happened because of bad update from a CRM vendor, would the headline say 'highlights MS CRM challenges?'"

Kurtz also confirmed Friday's bug "was not a cyberattack."
